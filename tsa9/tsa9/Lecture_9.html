1 Модель передаточной функции

1.1 Кросс-ковариационная, кросс-корреляционная функция

Пусть \(x_{t},y_{t},t=0,±1,±2,...,\) - стационаpные случайные пpоцессы в шиpоком смысле, т.е. \[Ex_{t}=μ_{x},Ey_{t}=μ_{y}\] \[Dx_{t}=σ_{x}²,Dy_{t}=σ_{y}²\] \[c_{x}(t,t+m)=E(x_{t}-μ_{x})(x_{t+m}-μ_{x})=c_{x}(m)\] \[c_{y}(t,t+m)=E(y_{t}-μ_{y})(y_{t+m}-μ_{y})=c_{y}(m)\] Кpосс-коваpиационная функция процессов \(x_{t},y_{t}\) опpеделяется фоpмулой \[c_{xy}(s,t)=E(x_{s}-μ_{x})(y_{t}-μ_{y}); s,t=0,±1,±2,...\] Cтационаpные случайные пpоцессы и называются совместно стационаpными в шиpоком смысле, если \[c_{xy}(t,t+m)=E(x_{t}-μ_{x})(y_{t+m}-μ_{y})=c_{xy}(m); m=0,±1,±2,...\]

Кpосс-коваpиационная функция паpы пpоцессов выpажает меpу линейной зависимости значений \(x_{t+m}и y_{t}\). Следует заметить, что кpосс-коваpиационная функция, вообще говоpя, не является симметpичной функцией аpгумента m. Веpно следующее соотношение \[c_{xy}(m)=c_{yx}(-m)\] Наиболее частое использование кpосс-коваpиационной функции иллюстpиpуется следующим пpимеpом. Пусть пpоцессы и связаны соотношением \(y_{t}=Ax_{t-d}\) где \(d\) - целое число. Говоpят, что \(x_{t}\) опеpежает \(y_{t }\) на \(d\) шагов, если \(d > 0\), и отстает от \(y_{t }\) на \(d\) шагов, если \(d < 0\). Пpедположим для пpостоты, что \(Ex_{t}=0,Ey_{t}=0\). Тогда кpосс-коваpиационная функция паpы пpоцессов \(x_{t}\) и \(y_{t}\) pавна \[c_{xy}(m)=Ex_{t+m}y_{t}=AEx_{t+m}x_{t-d}=Ac_{x}(m+d)\]

автоковаpиационной функции пpоцесса \(x_{t}\), умноженной на \(А\) и сдвинутой на \(|d|\) вpеменных лагов впpаво, если \(x_{t}\) опеpежает \(y_{t}\) на \(d\) шагов \((d > 0)\), или влево, если \(x_{t}\) отстает от на \(d\) шагов.

Продемонстрируем это свойство автокорреляции на процессе авторегрессии первого порядка

fi <- 0.9
sigma <- 2
model <- list(ar = fi)
x <- arima.sim(model=model,n = 200,innov=rnorm(200,0,sd=sigma))
acf(x,main = "ACF of x_t process",lwd = 5,col = "blue")

d <- 5
y <- x[d:200]
z <- x[1:(200-d+1)]
cross <- ccf(y,z,lwd = 5, main = "Cross-correlation of x_t and y_t",col = "blue")

1.2 Передаточные функции

Модели передаточной функции описывают динамическую взаимосвязь между выходом \(Y_t\) и одним или несколько входов \(X_{j,t}\). Особенно большой интерес эти модели имеют в тех случаях, когда выходная переменная явно зависит от входных переменных с некоторым запаздыванием. Расскажу историю из своей практики. В начале нулевых годов некоторая фирма, проводящая маркетинговые исследования на заказ обратилась ко мне помочь провести исследование собраных данных описывающих влияние рекламной компании на продажи пива. В качестве входных переменных \(X_{j,t}\) были количество денег, затраченных на рекламу в неделю, число минут рекламного времени по телевизору в неделю и т.п. Выходными переменными были итоги различных опросов: знают не знают сорт , употребляют ли регулярно и тому подобное, что конечно влияют на продажи за неделю конкретного сорта пива.

Важную роль в определении передаточной функции играет так называемая Функция отклика на единичный импульс (англ.the impulse response function (IRF)), которая для каждой входной переменной \(X_{j,t}\) имеет вид бесконечного полинома \[\nu_j(B)=\nu_{0,j}+\nu_{1,j}B+\nu_{2,j}B^2+....\] часто ее можно представить виде отношения конечных полиномов \[\nu_j(B)=\omega_{j,0}\frac{\omega_{j,s_j}(B)}{\delta_{j,r_j}(B)}B^{b_j}\] где \(s_j,r_j\) - степени полиномов в числителе и знаменателе,

\[\omega_{j,s_j}(B)= 1 -\omega_{j,1}B-\omega_{j,2}B^2-...-\omega_{j,s_j}B^{s_j}\] \[\delta_{j,r_j}(B)= 1 -\delta_{j,1}B-\delta_{j,2}B^2-...-\delta_{j,r_j}B^{r_j}\] в некоторых выдачах множитель \(\omega_{j,0}\) перед функцией в числителе \(\omega_{j,s_j}(B)\) вносят внутрь функции, перемножая \(\omega_{j,0}\omega_{j,s_j}(B)\) и после переобозначения коэффициентов произведения записывают \[\hat\omega_{j,s_j}(B)= \omega_{j,0} -\hat\omega_{j,1}B-\hat\omega_{j,2}B^2-...-\hat\omega_{j,s_j}B^{s_j}\]

Объединяя вышеприведенные выражения , мы получаем достаточно богатый класс линейных моделей TF (transfer function) для выхода \(Y_t\), заданного через функции импульсного откликов (IRF) для \(k\) входов \(X_{j,t},j=1,...,k\).

\[Y_t=\sum_{j=1}^k\omega_{j,0}\frac{\omega_{j,s_j}(B)}{\delta_{j,r_j}(B)}B^{b_j}X_{j,t}+N_t\] \[\delta_d(B)N(t)=\frac{\theta_q(B)}{\phi_p(B)}a_t\] где шум \(N_t\) следует общей модели ARIMA(p, d, q).

Определим задержку по времени между входом и выходом, вычислив кросс-корреляционную функцию входа и выхода meтодом из library(tfarima), который несколько отличен от стандартного ccf(). Поэтому ее и называют Prewhitened cross correlation function т.е. отбеленная кросс-корреляционная функция.

pccf(X, Y, um.x = umx, um.y = NULL, lag.max = 16)

где

  • X и Y — вход и выход,
  • um.x и um.y — используемые одномерные ARMA модели, чтобы предварительно отбелить оба ряда X и Y,
  • lag.max — это количество корреляций в каждой стороне.

Если оба um.х и um.y равны NULL, кросс-ковариационная функция оценивается без предварительного отбеливания и ее результат совпадет с методом со стандартным ccf()

ccf(X,Y,lag.max =16 )

pccf(X, Y, um.x = NULL, um.y = NULL, lag.max = 16)

Eсли только один из этих аргументов равен NULL, предоставленный um используется для предварительного отбеливания обоих рядов. В случае предоствления двух ARMA моделей um каждый ряд предварительно отбеливается своим um.

С идентификацией порядков модели передаточной функции сложнее. Просто предположим что порядки полиномов числителя и знаменателя равны 2. Потом проведем коррекцию порядков модели, если потребуется.

Итак, предположим, что модель такая \[Y_t=\frac{\omega_0-\omega_1B-\omega_2B^2}{1-\delta_1B-\delta_2B^2}B^5X_t+N_t\] \[N_t=\frac{1}{1-\phi_1B-\phi_2B^2-\phi_3B^3}a_t\]

tfx <- tfest(Y, X, delay = 5, p = 2, q = 2, um.x = umx, um.y = umy)

Оценим предложенную модель

tfmy <- tfm(Y, inputs = tfx, noise = um(ar = 3))

Посмотрим на коэффициенты \(\omega_0,\omega_1,\omega_2\). При этом будем использовать метод printLagpol(), который выдаст явный вид соответствующего полинома.

1.3 Прогнозирование по TF

Итак, будем считать, что модель наилучшем способом оценена и она содержится в переменной tfmy. Традиционно в R метод predict() осуществляет прогноз по модели, начиная прогноз с 5 наблюдения от конца ряда Y на n.ahead = 10 единиц времени вперед. Доверительный интервал для прогноза задан уровня 0.95.

nY <- length(Y)
forec <- predict(tfmy, n.ahead = 10, ori = nY-5,level = 0.95 )
## Warning in NextMethod("[<-"): number of items to replace is not a multiple of
## replacement length
plot(forec, n.back = 15, ylab = expression("Y"["t"]))

Если опустить параметр ori, то прогноз строится только для будущих значений ряда

forec <- predict(tfmy, n.ahead = 10,level = 0.95 )
plot(forec, n.back = 15, ylab = expression("Y"["t"]))
